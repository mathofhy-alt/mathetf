# 모의고사 공통 문항(가형/나형 등) 중복 처리 전략

## 1. 목적
가형, 나형 등에서 완전히 동일하게 출제된 수학 문제들이 중복되어 유저 경험을 해치거나 통계가 분산되는 것을 막기 위한 데이터베이스 설계 전략.

## 2. 채택된 아키텍처: '해시 기반 중복 삽입 허용 + 연동 ID(Hash Grouping)'
기존의 플랫(Flat)한 데이터 파이프라인(Next.js API -> Supabase) 구조를 해치지 않으면서, 100% 자동화로 묶어주는 방식을 채택함.

## 3. 향후 구현 방법 (To-Do)
나중에 "가형/나형 해시 중복 처리 적용해줘"라는 요청이 들어오면 다음 2가지를 작업할 것:

1. **DB 스키마 추가 (Supabase `questions` 테이블)**
   - `content_hash` (text): 문제 본문과 수식을 기반으로 한 SHA-256 해시값
   - `group_id` (uuid/int): 원본(최초) 문제의 ID (동일한 문항일 경우 동일한 group_id를 가짐)

2. **자동화 로직 끼워넣기 (API / Python 단)**
   - 문제를 데이터베이스에 `insert` 하기 직전, 문제의 텍스트(공백, 줄바꿈 제거)를 해싱.
   - DB에서 동일한 `content_hash`가 있는지 검색.
   - **없으면:** 일반 insert 후 자신의 ID를 `group_id`에 저장.
   - **있으면:** 일반 insert 하되, 검색된 원본 레코드의 ID를 찾아 `group_id`에 넣어서 연결(Linked)시킴.

## 4. 기대 효과
- 대표님의 한글 파일 기반 추출/업로드 워크플로우에 아무런 변화가 없음(추가 작업 불필요).
- 가형 15번과 나형 16번이 데이터상으로는 개별 존재하여 "시험지별 보기"에 완벽 대응.
- 서비스 UI에서 "이 문제는 나형에도 출제됨" 표시를 하거나 오답률 통계를 합산할 때 `group_id`로 손쉽게 묶을 수 있음.
